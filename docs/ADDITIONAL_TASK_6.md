# Дополнительное адание 6 — Страницы в формах (разрывы страниц)

Документ описывает изменения, внесённые в рамках реализации задания 6 (коммиты 6–10).

## Цель

Добавить возможность разбивать форму на страницы по аналогии с Google Forms: редактор получает кнопку вставки разрыва, а при заполнении формы поля отображаются постранично с навигацией «Назад / Далее».

---

## Изменённые файлы

### `src/server.js`

- В константу `ALLOWED_FIELD_TYPES` добавлен тип `page_break`. Без этого сервер при валидации входящих данных заменял бы тип на `text`, уничтожая разметку страниц при сохранении формы.

### `public/data.js`

- В массив `FORM_FIELD_TYPES` добавлен элемент `{ value: 'page_break', label: 'Разрыв страницы' }` — он используется в выпадающих списках типов полей во всех редакторах.
- В функции `addFormHistory` поле `fieldsCount` теперь считает только содержательные поля: `f.fields.filter(x => x.type !== 'page_break').length`. Разрывы страниц не являются вопросами и не должны учитываться в статистике.

### `public/index.html`

- В блок редактирования формы (`#form-edit-block`) добавлена кнопка `#form-add-page-break-in-edit` — «+ Разрыв страницы».
- В конструктор новой формы (`#form-fields-list`) добавлена кнопка `#add-form-page-break` — «+ Разрыв страницы».
- Перед кнопкой «Отправить ответ» добавлен контейнер `#form-page-nav` для динамической навигации по страницам.

### `public/app.js`

- Добавлена переменная `formPreviewPage` (начальное значение `0`) — хранит индекс текущей страницы предпросмотра.
- В `openFormView` добавлен сброс `formPreviewPage = 0` при открытии формы.
- В обработчике вкладок формы: при переключении на «Предпросмотр» `formPreviewPage` сбрасывается в `0` и вызывается `renderFormPreviewBlock()`.
- В fallback-массив `FORM_FIELD_TYPES_ARR` добавлен `page_break`.
- Функции `renderFormEditBlock` и `renderFormBuilderFields` дополнены веткой для `page_break`: вместо стандартных элементов управления (тип, подпись, обязательность) отрисовывается компактный разделитель `form-field-page-break` с текстом «— Разрыв страницы —» и кнопкой удаления ✕.
- Добавлены обработчики кнопок `#form-add-page-break-in-edit` и `#add-form-page-break`: добавляют в список полей запись `{ type: 'page_break' }`.
- Функция `renderFormPreviewBlock` переписана:
  - Поля формы разбиваются на страницы по `page_break`-разделителям (сами разделители в страницы не включаются).
  - Все страницы рендерятся в DOM как `<div class="form-page">`, неактивные скрыты через `style="display:none"` — значения полей сохраняются при навигации.
  - При наличии нескольких страниц в `#form-page-nav` динамически вставляются кнопки «← Назад» / «Далее →» и счётчик «Страница X из N».
  - Кнопка «Отправить ответ» скрыта на всех страницах кроме последней.
- В обработчике отправки ответа (`#form-submit-response`) поля типа `page_break` отфильтрованы: `normalizeFormFields(form).filter(fld => fld.type !== 'page_break')`. Это исключает разрывы из сбора ответов и проверки обязательных полей.
- В `renderFormResponsesBlock` аналогичная фильтрация исключает `page_break` из заголовков и ячеек таблицы ответов.

### `public/styles.css`

- `.form-field-page-break` — стиль карточки разрыва в редакторе: пунктирная граница, нейтральный фон, выравнивание по краям.
- `.page-break-divider` — стиль текста «— Разрыв страницы —»: по центру, приглушённый цвет.
- `.form-page-nav` — flex-контейнер навигации с отступами.
- `.form-page-counter` — счётчик страниц по центру, приглушённый цвет.

---

## Результат

В редакторе и конструкторе форм появилась кнопка «+ Разрыв страницы». При заполнении формы с разрывами пользователь видит поля постранично, перемещается кнопками «Назад» / «Далее» и отправляет ответ только с последней страницы. Разрывы не попадают в таблицу ответов и не учитываются в счётчике полей журнала действий.
