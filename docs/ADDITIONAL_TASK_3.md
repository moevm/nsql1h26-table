# Дополнительное задание 3 — Импорт таблиц из XLS

Документ описывает изменения, внесённые в рамках реализации задания 3.

## Цель

Добавить возможность импортировать таблицы из файлов формата `.xls`.

---

## Изменённые файлы

### `public/index.html`

- В выпадающий список формата импорта `#import-format` добавлен вариант `XLS`.
- В поле выбора файла `#import-file` добавлено расширение `.xls` в атрибут `accept`.
- Пользователь может выбрать файл `.xls` через стандартный диалог выбора файла.

### `public/app.js`

- Функция `detectDataFileFormat()` теперь распознаёт расширение `xls`.
- Обработчик кнопки импорта передаёт файлы форматов `xls`, `xlsx` и `ods` в общий модуль `window.AppSpreadsheet.parseSpreadsheetData()`.
- Для `.xls` используется `FileReader.readAsArrayBuffer()`, потому что настоящий XLS является бинарным файлом.
- CSV/XML-импорт дополнен начальными полями `sheets` и `activeSheetId`, чтобы импортированные таблицы были совместимы с новой системой вкладок.

### `public/spreadsheet.js`

- Добавлена поддержка чтения настоящих XLS-файлов:
  - разбор Compound File Binary Format (CFB), в котором хранится классический `.xls`;
  - поиск потока `Workbook` / `Book`;
  - чтение BIFF-записей;
  - извлечение строк, ячеек, чисел, строковых значений и дат;
  - поддержка общих строк через SST;
  - fallback для HTML-таблиц, сохранённых с расширением `.xls`.
- Функция `parseSpreadsheetData()` теперь выбирает парсер по формату:
  - `xlsx` — XLSX-парсер;
  - `xls` — новый XLS-парсер;
  - `ods` — ODS-парсер.
- При импорте обычной spreadsheet-книги видимые листы собираются в одну таблицу с вкладками, а не создаются как разрозненные таблицы.
- Для каждого листа создаётся вкладка с собственными `id`, `name` и `cells`.
- Если в файле присутствует служебный лист `_AppData`, импортируется сохранённое состояние приложения, а видимые листы применяются к таблицам как вкладки.

### `public/data.js`

- Нормализация таблиц поддерживает импортированные данные с `sheets` и `activeSheetId`.
- Старый формат с одним массивом `cells` автоматически преобразуется в таблицу с одной вкладкой.

### `src/server.js`

- Серверная нормализация таблиц поддерживает импортированные вкладки.
- При сохранении состояния импортированные листы раскладываются в `tableRows` с учётом `sheetId`, `sheetName` и `sheetOrder`.
- Старые таблицы без вкладок автоматически получают вкладку по умолчанию `Лист 1`.

---

## Результат

В интерфейсе импорта появился формат `XLS`, а приложение научилось принимать файлы `.xls` напрямую. XLS-файл читается как бинарный spreadsheet-файл, его листы импортируются в одну таблицу как вкладки.
